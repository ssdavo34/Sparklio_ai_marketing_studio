## 📊 Executive Summary

### 핵심 발견 사항

⚠️ **중요**: Polotno SDK는 **무료 플랜이 존재하지 않습니다**

**SDK vs Studio 구분**:
- **Polotno SDK**: 개발자용 라이브러리 (유료 전용)
- **Polotno Studio**: 크리에이터용 웹 앱 (무료 사용 가능)

### 가격 정보 (2025년 11월 기준)

| 플랜 | 월 비용 | 주요 특징 |
|------|---------|----------|
| **Team** | $155.83/월 | 기본 SDK 기능 |
| **Business** | $332.50/월 | 고급 기능 + 우선 지원 |
| **Enterprise** | 맞춤 견적 | 온프레미스, 화이트라벨 |

**무료 체험**: ✅ 개발 서버에서 100일 무료 체험 가능

## 🔍 상세 조사 내용

### 1. 라이선스 구조

#### SDK (개발자용)
```yaml
Polotno SDK:
  목적: React/Next.js 애플리케이션에 에디터 통합
  라이선스: 상업용 유료 라이선스
  무료 체험: 100일 (개발 서버 전용)

  제약사항:
    - 프로덕션 배포 시 유료 플랜 필수
    - 개발 서버에서만 무료 사용 가능
    - 워터마크 정책: 명시적 정보 없음 (웹 조사 필요)
```

#### Studio (웹 앱)
```yaml
Polotno Studio:
  목적: 일반 사용자를 위한 디자인 도구
  라이선스: 무료 (기본 기능)
  유료 옵션: AI 기능은 크레딧 기반 과금

  특징:
    - 웹 브라우저에서 바로 사용
    - 별도 설치 불필요
    - 우리 프로젝트와는 무관
```

### 3. 무료 체험 정책

#### 개발 서버 100일 무료 체험

**조건**:
```yaml
무료 체험:
  기간: 100일
  환경: 개발 서버 전용 (localhost, dev.example.com 등)
  제약:
    - 프로덕션 도메인 사용 불가
    - 상업적 배포 불가

  확인 필요:
    - 워터마크 표시 여부?
    - 기능 제한 여부?
    - API 호출 제한?
```

**우리 프로젝트 적용**:
```yaml
현재 상황:
  - 개발 환경: localhost:3000 (무료 체험 사용 가능 ✅)
  - 테스트 기간: 11/22 ~ 12/26 (35일 < 100일 ✅)

예상 시나리오:
  1. 개발 단계 (현재 ~ 12/26):
     - 무료 체험 사용 ✅
     - 워터마크 확인 필요 ⚠️

  2. 프로덕션 배포 (12/26 이후):
     - 유료 플랜 필수 🔴
     - 최소 Team 플랜 ($155.83/월)
```

### 4. 워터마크 정책 (조사 필요)

⚠️ **공식 문서에서 명확한 정보 미확인**

**가능성 1**: 무료 체험 시 워터마크 표시
```yaml
예상:
  - 위치: 우측 하단 "Powered by Polotno"
  - 크기: 작은 로고 + 텍스트
  - 제거: 유료 플랜 가입 후 제거
```

**가능성 2**: 무료 체험 시 워터마크 없음
```yaml
예상:
  - 개발 환경에서는 워터마크 없음
  - 프로덕션 배포 시에만 라이선스 체크
  - 라이선스 없으면 에러 또는 워터마크
```

**확인 방법**:
1. ✅ **직접 테스트** (최우선)
   - localhost에서 Polotno 설치
   - 캔버스 렌더링 후 워터마크 확인
   - 내보내기 시 워터마크 확인

2. 공식 문서 추가 조사
   - https://polotno.com/docs
   - GitHub Issues 검색

3. 커뮤니티 문의
   - Polotno Slack/Discord
   - Stack Overflow 검색

### 6. 리스크 평가

#### 🔴 High Risk

**1. 라이선스 비용 증가**
```yaml
리스크:
  - Business 플랜 필요 시 월 $332.50 (2배 증가)
  - Enterprise 플랜 필요 시 예산 초과 가능

완화 방안:
  - Team 플랜으로 시작
  - 필요 시에만 Business로 업그레이드
  - 초기 계약 시 할인 협상
```

**2. 라이선스 종속성**
```yaml
리스크:
  - Polotno 서비스 중단 시 전체 시스템 영향
  - 가격 정책 변경 가능성

완화 방안:
  - 연간 계약으로 가격 고정
  - 향후 자체 엔진 전환 계획 수립 (2년 후)
```

#### 🟡 Medium Risk

**3. 기능 제약**
```yaml
리스크:
  - Polotno가 지원하지 않는 기능 구현 불가
  - 커스터마이징 한계

완화 방안:
  - 사전에 필요 기능 리스트 확인
  - Polotno API 확장성 검토
  - 필요 시 Hybrid 방식 (Polotno + Custom Canvas)
```
