- 슬라이드마다 **비주얼 디자인**이 필요

- **발표자 노트**까지 고품질로 작성해야 함
- **애니메이션/전환 효과** 고려 필요

## 🎯 제안: 프리젠테이션 에이전트 파이프라인

고품질 프리젠테이션을 위해 **멀티 에이전트 파이프라인**이 필요합니다:

```
┌─────────────────────────────────────────────────────────────────┐
│                    프리젠테이션 생성 파이프라인                    │
└─────────────────────────────────────────────────────────────────┘

Step 1: [PresentationStrategist]
        └─ 전체 구조 설계 (슬라이드 개수, 흐름, 핵심 메시지)
                    ↓
Step 2: [PresentationWriter]  ← Claude 3.5 Haiku
        └─ 슬라이드별 텍스트 콘텐츠 작성
           (제목, 본문, 발표자 노트)
                    ↓
Step 3: [PresentationDesigner]
        └─ 슬라이드별 비주얼 프롬프트 생성
           (레이아웃, 이미지, 차트, 색상)
                    ↓
Step 4: [VisionGenerator] (기존)
        └─ 실제 이미지 생성
                    ↓
Step 5: [PresentationReviewer]
        └─ 품질 검토 및 개선 제안
```

### 각 에이전트 역할

|에이전트|역할|LLM|
|---|---|---|
|**PresentationStrategist**|청중 분석, 스토리 흐름, 슬라이드 구조|Claude Haiku|
|**PresentationWriter**|슬라이드별 텍스트, 발표자 노트|Claude Haiku|
|**PresentationDesigner**|비주얼 가이드, 이미지 프롬프트|Claude Haiku|
|**VisionGenerator**|실제 이미지 생성|ComfyUI/DALL-E|
|**PresentationReviewer**|품질 검토, 일관성 체크|Claude Haiku|

|**VisionGenerator**|프롬프트 → 이미지 생성|❌ 없음|
|**Copywriter**|슬라이드 텍스트/발표자 노트|⚠️ 약간 (presentation 작업 추가)|
|**Editor**|텍스트 교정/번역|❌ 없음|
|**Strategist**|슬라이드 구조 기획|⚠️ 약간 (presentation 작업 추가)|
|**Reviewer**|품질 검토|⚠️ 약간|

## 🎯 제안: 프리젠테이션 멀티 에이전트 파이프라인

현재 `PresentationAgent`가 혼자 모든 걸 하는 대신, **기존 에이전트들을 조합**:

```
┌─────────────────────────────────────────────────────────────────┐
│              Sparklio 프리젠테이션 생성 파이프라인                 │
└─────────────────────────────────────────────────────────────────┘

Step 1: Strategist (구조 기획)
        └─ 슬라이드 개수, 흐름, 핵심 메시지 결정
        └─ LLM: Claude 3.5 Haiku
                    ↓
Step 2: Copywriter (텍스트 작성)  
        └─ 슬라이드별 제목, 본문, 발표자 노트
        └─ LLM: Claude 3.5 Haiku
                    ↓
Step 3: VisualPrompt (프롬프트 생성) ← 이미 구현됨!
        └─ 컨셉 → 영문 이미지 프롬프트
        └─ LLM: Gemini Flash (빠름)
                    ↓
Step 4: VisionGenerator (이미지 생성) ← 이미 구현됨!
        └─ 프롬프트 → 실제 이미지
        └─ Nanobanana/ComfyUI
                    ↓
Step 5: Reviewer (품질 검토) [선택]
        └─ 텍스트/이미지 검토
```

---

## 💡 이점

|현재 (단일 에이전트)|제안 (멀티 에이전트)|
|---|---|
|PresentationAgent 혼자 처리|전문 에이전트 분업|
|이미지 생성 없음|VisionGenerator로 실제 이미지|
|6-10장 한계|12-15장+ 가능|
|품질 검토 없음|Reviewer로 품질 보장|
