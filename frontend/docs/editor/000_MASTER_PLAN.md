## 핵심 차별점: AI + Learning + Automation

### 🎯 Sparklio의 3대 핵심 기능 (다른 에디터와의 결정적 차이)

#### 1. Meeting AI Integration (진입 장벽 제거)

**문제**: 마케팅 에셋 제작은 시간이 오래 걸리고 전문가가 필요함

**해결**:
- 회의록 한 장 → 5분 만에 완성된 초안
- 디자인 경험 없어도 전문가급 결과물
- 회의 내용이 자동으로 역할별 객체에 매핑

**기술 스택**:
```typescript
// Meeting AI → Editor 연동 프로토콜
interface MeetingToEditorCommand {
  type: "GENERATE_FROM_MEETING";
  payload: {
    meetingId: string;
    meetingSummary: {
      contentType: "product-detail" | "pitch-deck" | "ad" | "blog";
      sections: {
        role: ObjectRole;          // "headline", "body", "cta-button"
        content: string;            // 실제 텍스트 내용
        priority: number;           // 중요도 (레이아웃 결정)
      }[];
    };
    brandContext?: {
      brandId: string;
      designTokens: DesignTokens;
    };
  };
}
```

#### 2. Admin Learning & Trend Engine (지속적 개선)

**문제**: 트렌드는 계속 변하는데 템플릿은 고정되어 있음

**해결**:
- 실시간 마케팅 데이터 크롤링 (인스타그램, 틱톡, 네이버 블로그)
- 성공 패턴 자동 학습 (CTR, CVR 기반)
- 템플릿 자동 생성 및 랭킹 업데이트

**관리자 모드**:

#### 3. Auto Template Generator (생산성 폭발)

**문제**: 같은 형식의 광고를 10개, 20개 만들어야 함 (A/B 테스트, 멀티 채널)

**해결**:
- 브랜드 + 트렌드 패턴 기반 템플릿 자동 생성
- 객체 레벨 편집 가능 (단순 이미지가 아님)
- 성능 데이터 기반 변형 추천

**자동 생성 예시**:
```typescript
// Request
POST /api/v1/templates/auto-generate
{
  "brandId": "nike-kr",
  "contentType": "instagram-story",
  "market": "kr",
  "count": 5,          // A/B 테스트용 5개 변형
  "trendPreference": "high"  // 최신 트렌드 우선
}

// Response: 5개의 EditorDocument (각각 편집 가능한 객체로 구성)
[
  {
    "id": "doc-1",
    "templateId": "trend-ig-story-001",
    "trendSnapshotId": "2025-11-kr-ig-story-snapshot-42",
    "pages": [{
      "objects": [
        {
          "type": "text",
          "role": "headline",
          "content": "Just Do It",  // 자동 생성되었지만 수정 가능
          "source": {
            "kind": "auto-generated",
            "trendId": "trend-ig-story-001"
          }
        },
        {
          "type": "image",
          "role": "product-image",
          "source": { "kind": "template" }
        }
      ]
    }]
  }
  // ... 나머지 4개 변형
]
```

## 개요

### 프로젝트 배경

Canvas Studio v3는 **Figma + Canva + Notion을 결합한 마케팅 전문 크리에이티브 스튜디오**입니다.

**현재 상황**:
- ✅ Konva.js 기반 CanvasEngine 구현 완료
- ✅ Zustand 기반 EditorStore 구현 완료
- ✅ 기본 드래그, 선택, 변형 기능 동작 중
- ✅ 레거시 Fabric.js 에디터 완전 제거
- ⚠️ 전문 에디터로서의 고급 기능 부족
- ⚠️ 다양한 콘텐츠 타입 지원 필요

### 비전

**"하나의 에디터에서 모든 마케팅 크리에이티브를 생성하고 관리한다"**

- 상품 상세페이지
- 피치덱 프레젠테이션
- SNS 광고 (Instagram, TikTok, YouTube)
- 블로그 포스트
- 이메일 템플릿
- 브랜드 에셋 관리

```
│    └─ Konva Layer                                           │
│         ├─ Text Node  (EditorStore의 Text Object 렌더링)   │
│         ├─ Image Node (EditorStore의 Image Object 렌더링)  │
│         └─ Shape Node (EditorStore의 Shape Object 렌더링)  │
│                                                              │
│  Events: click, drag, transform → EditorStore 업데이트     │
└─────────────────────────────────────────────────────────────┘
```

### 데이터 흐름

1. **사용자 액션** → CanvasEngine (Konva Event)
2. **CanvasEngine** → EditorStore (State Update)
3. **EditorStore** → React Re-render
4. **React** → CanvasEngine (Konva Node Update)

**핵심 원칙**:
- ✅ Konva는 렌더링만 담당
- ✅ 모든 상태는 EditorStore에만 존재
- ✅ Konva는 EditorStore를 구독하여 동기화
