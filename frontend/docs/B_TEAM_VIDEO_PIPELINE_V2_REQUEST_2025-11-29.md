## 핵심 변경사항 요약

### 1. PLAN/RENDER 2단계 분리

```
기존: 이미지 선택 → 바로 영상 생성
신규: 이미지 선택 → [PLAN] 스크립트 확인/수정 → [RENDER] 영상 생성
```

**장점**: 비용 절감 (유저가 확인 후에만 렌더링)

### 2. 3가지 이미지 모드

| 모드 | 설명 | 비용 |
|------|------|------|
| REUSE | 기존 이미지만 사용 | 무료 |
| HYBRID | 일부 재사용 + 일부 신규 | 중간 |
| CREATIVE | 전부 신규 생성 | 높음 |

### 3. 신규 API 엔드포인트

| Method | Path | 용도 |
|--------|------|------|
| POST | /api/v1/video6/{id}/plan | 스크립트 초안 생성 |
| PUT | /api/v1/video6/{id}/plan | 유저 수정본 저장 |
| POST | /api/v1/video6/{id}/render | 영상 렌더링 시작 |
| GET | /api/v1/video6/{id}/status | 상태 조회 |
| GET | /api/v1/video6/{id}/assets | Asset Pool 조회 |
