**작성팀**: C팀 (Frontend Team)

## 1. Executive Summary

본 보고서는 Sparklio AI Marketing Studio의 UI 전면 개편에 대한 종합 검토 결과입니다.

### 핵심 개편 방향
1. **좌측 메뉴 재구조화**: 기능 중심에서 워크플로우 중심으로 전환
2. **상단 플로우 바 도입**: 작업 진행 상태 시각화
3. **전역 작업 센터**: 백그라운드 작업 관리 시스템
4. **Undo/Redo 히스토리**: 메뉴별 독립 히스토리 스택

### 개편 효과
- 회의 → 컨셉 → 산출물까지 **끊김 없는 워크플로우**
- 작업 중 다른 메뉴 이동 시에도 **상태 유지**
- AI 생성 작업의 **비동기 처리 + 알림 시스템**
- 실수해도 **언제든 되돌릴 수 있는** 안전한 작업 환경

### 3.1.1 Assets (에셋 라이브러리) 역할

**정의**: 여러 에이전트와 메뉴에서 만들어진 산출물을 한 번에 모아 보고, 다시 열어보고, 재활용하는 **"결과물 라이브러리"**

**포함 콘텐츠**:
- 프리젠테이션 초안
- 상세페이지 초안
- SNS 광고 카피/레이아웃
- 쇼츠 스크립트/썸네일
- 배너 이미지
- 생성된 모든 비주얼 에셋

**핵심 기능**:
| 기능 | 설명 |
|------|------|
| **통합 리스트** | 모든 채널에서 만들어진 결과물 한 곳에서 조회 |
| **필터링** | 채널 / 콘셉트 / 날짜 / 상태(초안/완료) |
| **퀵 액션** | 클릭 시 해당 채널 에디터로 이동 |
| **재활용** | 기존 에셋을 새 프로젝트에 복제/활용 |
| **백그라운드 연결** | 작업 완료 시 자동으로 리스트에 추가 |

**워크플로우 연결**:
```
[ConceptBoard에서 "풀셋 생성"]
         ↓
[Background Task Center에서 진행]
         ↓
[완료 시]
├── 각 채널 메뉴에 나타남 (Presentation, SNS 등)
└── Assets 리스트에도 새 항목으로 추가 ← 여기서 통합 관리
```

**사용 시나리오**:
- "최근 1주일 간 만든 산출물만 보고 싶다" → Assets 열고 날짜 필터
- "A 콘셉트로 만든 모든 결과물 확인" → Assets 열고 콘셉트 필터
- "지난주 만든 인스타 광고 재활용" → Assets에서 찾아서 복제

┌──────────────────────────────────────────────────────────────────────────────┐
│ [Project ▼]  [플로우 바━━━━━━━━━━━━━━━━━━]  [⟲2] [Undo] [Redo] [저장됨] [Export] │

#### 플로우 바 상세

```
[Brand Kit] ─ [Meeting AI] ─ [ConceptBoard] ─ [Presentation] ─ [상세] ─ [SNS] ─ [영상] ─ [이미지] ─ [Assets]
    ●             ◐              ◐               ○            ○       ○       ○        ○         ◐

● = Ready (완료)
◐ = Draft (작업 중) - Assets는 생성된 항목 있으면 Draft
○ = Off (미시작)
```

**Assets 상태 표시 로직**:
- ○ Off: 아직 생성된 에셋 없음
- ◐ Draft: 1개 이상 에셋 존재
- ● Ready: 모든 채널 산출물 완료

### 3.4 Undo/Redo 히스토리 시스템

#### 히스토리 대상 액션

| 메뉴 | Undo 대상 |
|------|----------|
| Brand Kit | 컬러 변경, 폰트 교체, 로고 교체, 톤앤매너 수정 |
| Meeting AI | 인사이트 카드 삭제/고정, ConceptBoard로 보내기 |
| ConceptBoard | 카드 생성/삭제/편집, 순서 변경, 풀셋 생성 |
| Presentation | 섹션 추가/삭제, 텍스트/이미지/색상 변경 |
| 상세페이지 | 섹션 편집, 이미지 교체, 레이아웃 변경 |
| SNS | 템플릿 교체, 텍스트 수정, 필터 적용 |
| Video | 씬 추가/삭제, 전환 효과 변경 |
| Image | 스타일 변경, 프롬프트 수정 |

#### 히스토리 Store 구조

```typescript
type HistoryState<T> = {
  past: T[];      // 이전 상태들
  present: T;     // 현재 상태
  future: T[];    // Redo용 상태들
};

interface EditorHistoryStore {
  brandKit: HistoryState<BrandKitState>;
  meeting: HistoryState<MeetingState>;
  conceptBoard: HistoryState<ConceptBoardState>;
  presentation: HistoryState<PresentationState>;
  detail: HistoryState<DetailState>;
  sns: HistoryState<SNSState>;
  video: HistoryState<VideoState>;
  image: HistoryState<ImageState>;

  activeEditor: EditorType;

  undo: () => void;
  redo: () => void;
  push: (editor: EditorType, state: any) => void;
  canUndo: () => boolean;
  canRedo: () => boolean;
}
```

## 4. 주요 플로우 설계

### 4.1 회의에서 시작하는 플로우

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            회의 기반 풀셋 생성 플로우                          │
└─────────────────────────────────────────────────────────────────────────────┘

[1] Meeting AI 진입
    │
    ├──▶ 회의 영상/오디오 업로드
    │
    ├──▶ 자동 전사 + 인사이트 추출 (백그라운드)
    │    └── 사용자는 다른 메뉴 작업 가능
    │
    └──▶ 완료 시: 토스트 알림 + 배지 표시

[2] 인사이트 확인
    │
    ├──▶ 요약 카드
    ├──▶ 액션 아이템 카드
    ├──▶ 콘셉트 후보 카드 (3개)
    └──▶ 프레젠테이션 아웃라인 카드

[3] ConceptBoard로 전송
    │
    └──▶ [→ 콘셉트 보드로 보내기] 클릭
         └── 선택한 콘셉트 카드들이 ConceptBoard에 생성

[4] 산출물 생성
    │
    ├──▶ 개별 생성: 슬라이드/상세/SNS/영상/이미지 각각 클릭
    │
    └──▶ 풀셋 생성: [풀셋 생성] 버튼
         └── 5개 채널 동시 생성 (백그라운드)
```

### 4.2 브랜드 킷에서 시작하는 플로우

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            브랜드 기반 콘텐츠 생성 플로우                        │
└─────────────────────────────────────────────────────────────────────────────┘

[1] Brand Kit 설정
    │
    ├──▶ 브랜드 컬러 팔레트
    ├──▶ 폰트 패밀리
    ├──▶ 로고 업로드
    └──▶ 톤앤매너 정의

[2] ConceptBoard에서 수동 생성
    │
    └──▶ 채팅으로 콘셉트 생성 요청
         └── Brand Kit 정보 자동 참조

[3] (선택) Meeting AI 보강
    │
    └──▶ 나중에 회의 자료 추가 시
         └── "기존 콘셉트와 비교해서 수정 제안해줘"
              └── 기존 카드 업데이트 or 새 카드 추가

[4] 산출물 생성
    │
    └──▶ 개별/풀셋 생성 (Brand Kit 스타일 자동 적용)
```

## 10. 결론

본 UI 개편은 Sparklio의 핵심 가치인 **"회의에서 콘텐츠까지 한 번에"**를 실현하기 위한 필수 작업입니다.

### 핵심 성과 지표

| 지표 | 현재 | 목표 |
|------|------|------|
| 회의 → 산출물 평균 클릭 수 | 12+ | 5 이하 |
| 작업 중 이탈률 | 높음 | 50% 감소 |
| "처음부터 다시" 발생 빈도 | 빈번 | 거의 0 |
| 백그라운드 작업 인지율 | 낮음 | 100% |

### 최종 권고

1. **Phase 1부터 즉시 착수**: Undo/Redo와 작업 센터는 UX의 기반
2. **점진적 마이그레이션**: 한 번에 전면 개편보다 단계적 적용
3. **사용자 피드백 수집**: Phase별 완료 후 내부 테스트 진행
